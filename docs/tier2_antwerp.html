Scratch storage upgrade

Start: 2026-03-18 09:00:00
End: 2026-03-18 14:00:00
Updated: 2026-03-17 08:30:57
Level: high

On Wednesday March 18, an upgrade of the scratch filesystem services will take place (to fix bugs in the current version of the services). The upgrade will start at 9 am and might take a few hours to complete.

During the upgrade, there might be small interruptions in the availability of the scratch filesystems (as these services need to be restarted).

Therefore, we have created a reservation on all compute nodes between 9 am and 2 pm. Jobs that cannot run to completion before the start of the upgrade will not start before the end of the reservation (2 pm).

Our apologies for the inconvenience.
